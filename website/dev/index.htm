Introduction to Glich Development

The program is currently being developed on the MS Windows OS, but since it is written in standard C++ (ISO C++17) it will be available on other operating systems.

The project is broken down into the following sub projects.

1. The Glich Language. glc.lib

This is the engine that forms the basis for subsequent extensions. These are named hics, higs and hils. These extensions are, or will be, part of the genealogy program The Family Pack.

As stated, the library is written in C++. It has only one external dependency, a small utf-8 C library, utf8api which is included in the project.

2. Glich Console Program. glcs.exe

A command line program which can be used to test out the language. This will be released soon to allow others to experiment with it.

3. The Test Scripts and Program. glctest.exe

A series of Glich Script files have been created using a standard comment format to indicate the expected output of the script. The test program then runs through each file and compares its output to that expected.

The Hics Extension

Hics is the HistoryCal extension to the language to allow calendar schemes to be defined and manipulated.

4. The Hics Unit Test Program. hicunit.exe

Proposed: A unit test program to test the interface between the HistoryCal GUI program and the Glich language library.

5. The Hics Unit Test Soak Program. hicsoak.exe

Proposed: A unit test program which runs exhaustive tests on specific calendar schemes. The program will run through every date for the defined usage range of the calendar scheme. The program is expected to take some time to run.
